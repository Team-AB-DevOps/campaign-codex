# Software Requirements Specification: Campaign Codex

## 1. Introduction

Campaign Codex is a web-based application designed to help Tabletop RPG (TTRPG) players and Game Masters (DMs) organize their games. The goal is to replace physical notebooks and loose papers with a single, easy-to-use digital platform. It allows users to track campaign stories, characters, and world details in one place.

## 2. Target Audience

-   **Game Masters (DMs):** Need tools to manage the world, plot, NPCs, and session notes.
-   **Players:** Need a place to keep their character sheets, inventory, and personal notes.

## 3. Functional Requirements

The system must support the following core features:

-   **Campaign Management:** Users can create and join campaigns. A campaign acts as a container for all game data.
-   **Character Tracking:** Players can create characters with stats, descriptions, and backstories. These characters are linked to specific campaigns.
-   **Wiki & Lore:** A flexible wiki system where DM's can write articles about locations, items, and history in campaigns. Linking between wiki entries should be possible.
-   **Session Notes:** A dedicated section for writing and organizing notes from game sessions.
-   **Interactive Map:** DM can upload map image. The system should support pinning locations on maps (map pins) with descriptions.
-   **Image Upload:** Users can upload images (For wiki entries or characters).
-   **User Accounts:** Secure login and registration to protect user data.

## 4. Non-Functional Requirements

-   **Simplicity:** The interface should be clean and intuitive, avoiding complex menus that slow down gameplay.
-   **Accessibility:** The app should be accessible via any standard web browser.
-   **Performance:** Pages should load quickly to ensure it doesn't disrupt the flow of the game.
